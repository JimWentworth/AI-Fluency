AI and the Student Learning Experience

Learning Cycle: Discover → Understand → Practice → Create → Reflect → Reiterate

UDL emphasis: this matrix expands multiple means of expression by proposing varied student artifacts across text, voice, visuals, interactive media, and data. Each cell lists 4–6 creative, discipline-flexible student actions and artifacts (not tool features). Voice and phone-based vision appear where they improve learning (oral rehearsal, transcript-based revision, field observation, annotated photos). The set is designed to reduce repetition by ensuring each cell spans distinct artifact types (e.g., dialogue scripts, case studies, datasets, websites, infographics, notebooks, audits).

Discover
  • Curiosity Map: generate 8 questions, then sort them into “definitions / mechanisms / evidence / implications.”
  • Translation Ladder: rewrite an idea for two audiences (novice and expert) and note what had to change.
  • Stakeholder Preview: simulate a stakeholder who asks “what matters here?” and capture the top concerns.
  • Voice Warmup: record a 60-second “what I think this unit is,” transcribe, and mark unknown terms.
  • Concept Boundary List: produce “what this is / what this is NOT” statements to guide initial reading.
  • Advance Organizer: outline the unit as a checklist of skills (“I should be able to…”).
  • Research Funnel: generate broad→narrow research questions with clear constraints (scope, time, evidence).
  • Comparison Scaffold: create a blank matrix to fill (theory/method/model vs criteria).
  • Starter Code Sketch: write pseudocode that represents a system/process (even if not executable yet).
  • Case Seed Prompts: draft prompts that could generate multiple relevant case studies later.
  • Concept Postcard: one-slide preview (headline, 3 claims, 1 diagram, 1 driving question).
  • Learning Hub Page: a simple webpage to store definitions, sources, and questions as you go.
  • Audio Preview: short narration that states what you expect to learn and what you’ll verify.
  • Mini Timeline: timeline slide/webpage for key developments (historical, procedural, conceptual).
  • Media Lens Check: compare how the same idea reads in text vs diagram vs short narration.
  • Phone Capture Inventory: photograph a real object/setup and label parts/features (no interpretation yet).
  • Figure Walk: list observations from a chart/figure (axes, units, patterns) before interpreting.
  • Alt-Text Draft: write alt-text for a figure and verify it matches what’s shown.
  • Mind Map Seed: generate nodes and link types (cause/effect/example/mechanism).
  • Infographic Plan: draft a layout for a one-claim infographic to populate later.
  • Dataset Recon: identify variables, units, missingness, and limits of inference.
  • Valid/Invalid Questions: generate questions the dataset can and cannot answer (with reasons).
  • Data Dictionary: draft definitions + assumptions checklist before analysis.
  • Bias Radar: list likely biases (sampling, measurement) and what patterns they’d produce.
  • Synthetic Data Sketch: describe what a “toy dataset” would look like if the theory is true.
Understand
  • Socratic Probe: give your explanation; answer 5 follow-up “why/how do you know?” questions.
  • Oral Exam Rehearsal: simulate an oral exam; transcript your answers and mark weak warrants.
  • Example/Non-example Set: generate examples and near-misses; justify classifications using criteria.
  • Role Explanation: explain the concept as a professional role (clinician/engineer/critic), then compare.
  • Misconception Refutation: identify a tempting wrong view and refute it with evidence.
  • Three Explanations: novice/technical/metaphor; critique each for accuracy and limits.
  • Concept-to-Code: write pseudocode or a short annotated script representing the mechanism.
  • Boundary Tests: generate edge cases; state what the concept/model predicts and why.
  • Structured Summary: write a claim–evidence–limitation paragraph (not a generic summary).
  • Mini Case Interpretation: generate a short case and explain how the concept applies.
  • Mini-Lesson: narrated slides (plus transcript) explaining one process end-to-end.
  • Interactive Explainer: webpage with steps, definitions, and “check yourself” prompts.
  • Teach-Back Loop: record → transcribe → revise → re-record to improve clarity.
  • Storyboard: plan a 60–90s explainer video (scenes → claims → evidence).
  • Concept Map Web: publish a simple concept map page with short link explanations.
  • Observation vs Inference: annotate a figure/photo with “O” and “I” tags.
  • Photo-to-Diagram: convert a real photo into a clean diagram with a legend.
  • Two-Model Compare: compare two visuals; explain which misleads less and why.
  • Mind Map Refinement: reorganize nodes into hierarchy and label link semantics.
  • Infographic Critique: evaluate claim–evidence fit and what the visual implies.
  • Interpretation Packet: interpret a plot/table + write “what it does NOT show.”
  • Methods Justification: justify a method choice by assumptions and question fit.
  • Competing Explanations: propose alternatives and specify what data would discriminate.
  • Toy Dataset Build: generate a small synthetic dataset consistent with a theory; describe expected patterns.
  • Qual Coding Start: draft a coding scheme for short texts and justify categories.
Practice
  • Hint Budget Log: one hint at a time after first attempt; document what changed and why.
  • Step Justification Drill: AI asks “why that step?” and you defend each move.
  • Role-Play Clinic: simulate a professional interaction; practice responses and debrief.
  • Argument Sparring: defend a claim; AI pushes counterpoints; you revise the warrant.
  • Pronunciation/Delivery Practice: record speaking, transcribe, mark clarity issues and revise script.
  • Fading Worked Examples: generate skeleton solutions with blanks that increase over time.
  • Micro-Code Drills: small functions/scripts that practice the same logic as the concept.
  • Unit Tests: test cases that validate outputs and catch edge cases.
  • Case Variants: generate multiple cases; solve; then classify by which principle applies.
  • Rubric Self-Check: score your attempt and identify one targeted improvement.
  • Annotated Walkthrough Video: record a solution demo with “why” callouts.
  • Interactive Tutorial: web steps + checkpoints + feedback for common errors.
  • Branching Scenario Practice: make a decision tree and justify each branch.
  • Mini Simulation Demo: simple web or slide-based simulation showing variables and outcomes.
  • Oral Rehearsal Loop: speak → transcribe → revise → re-record for coherence.
  • Phone Debug Photo: photograph your setup/work and run a checklist (missing parts, conventions, safety/quality).
  • Correct vs Incorrect Visuals: compare two diagrams/photos and diagnose the critical difference.
  • Annotation Practice: label a visual with criteria-based notes (what feature supports which claim).
  • Mind Map From Memory: redraw a mind map without notes; compare to reference; repair gaps.
  • One-Claim Infographic: produce claim + evidence + limitation + source note.
  • Cleaning Log: transform data with an audit trail + validation checks.
  • Chart Choice Drill: make 3 charts for the same data; justify best choice and risks.
  • Synthetic Dataset Practice: generate biased vs unbiased versions; predict how conclusions shift.
  • Notebook Practice: implement analysis in a reproducible notebook with narrative.
  • Sensitivity Check: run a simple robustness test and interpret conclusion stability.
Create
  • Stakeholder Panel: simulate stakeholders who interrogate a proposal; revise based on objections.
  • Debate Script: claims/warrants/rebuttals written to fit evidence standards.
  • Interview Artifact: create an interview (questions + follow-ups) and a coded transcript summary.
  • Policy/Design Pitch: create a persuasive pitch tailored to a specific audience and constraints.
  • Dialogue Exhibit: a Q&A-based artifact where questions drive the explanation.
  • Computational Artifact: prototype/app/script/notebook demonstrating the concept or method.
  • Docs Package: README + usage examples + limitations + ethical/assumption notes.
  • Testing Suite: automated tests that validate logic and edge cases.
  • Structured Argument Build: produce an argument map (claims → evidence → warrants).
  • Case Study Generator: create a set of cases plus an answer key/rubric for peer use.
  • Website Deliverable: interactive explainer site or digital exhibit with citations.
  • Micro-Course: short learning module with media + checks + feedback built by students.
  • Branching Case: interactive decision scenario with consequences and reflection prompts.
  • Tutorial Series: multi-part how-to with checkpoints and common-error guidance.
  • Podcast/Video Essay: argument + evidence + counterargument in a narrated format.
  • Photo-to-Infographic: convert field/lab photos into a clear infographic with caveats and sources.
  • Model Diagram Set: create diagrams + legends + “misread risk” notes.
  • Visual Standards Guide: define what “good” visuals look like in the discipline.
  • Synthetic Illustration: labeled synthetic visuals for abstract concepts (transparency required).
  • Annotated Field Notebook: photo + annotation + claim + evidence + limitation per entry.
  • Synthetic Dataset Build: design datasets to test a hypothesis (including bias scenarios).
  • Reproducible Analysis: notebook/script + outputs + narrative interpretation.
  • Dashboard: evidence summary with provenance notes and limitations.
  • Robustness Pack: alternative analyses and what changes (or doesn’t).
  • Data Story: question → method → results → uncertainty → implications.
Reflect
  • Interview Reflection: AI interviews you about a specific artifact; you cite evidence from your work.
  • Transcript Annotation: annotate an oral explanation transcript for weak warrants and missing evidence.
  • Decision Diary: explain 3 key decisions you made and what evidence informed each.
  • Misconception Autopsy: describe a wrong model you held and what corrected it.
  • Next-Step Commitments: set measurable goals and define the next artifact you’ll produce.
  • Synthesis Memo: connect concepts; explicitly state limits and assumptions.
  • Claims Audit: list claims you made; attach the evidence source for each.
  • Code Reflection: explain why your solution is structured this way and what could break it.
  • Revision Plan: identify 2 rubric criteria to improve and concrete edits to make.
  • Question Audit: what you can answer confidently vs what remains uncertain.
  • Portfolio Website: curate artifacts with brief reflections on growth and tradeoffs.
  • Narrated Walkthrough: explain decisions with evidence callouts (audio/video supported).
  • User Confusion Review: analyze where peers got stuck in your interactive artifact and why.
  • Accessibility Reflection: add captions/transcripts/alt-text and reflect on impact.
  • Media Compare: reflect on what you express best in audio vs text vs interactive web.
  • Before/After Visual Audit: compare early vs late visuals; identify improvement patterns.
  • Misread Prediction: predict viewer misreads; revise visuals to reduce ambiguity.
  • Phone Field Reflection: pick 3 photos and explain what each taught you and what remains uncertain.
  • Transparency Note: document where synthetic visuals were used and why appropriate.
  • Standards Checklist: personal checklist for future visuals (conventions + clarity).
  • Error Pattern Report: analyze mistakes across work and propose targeted interventions.
  • Conclusion Stability: reflect on what would change your conclusion (and why).
  • Reproducibility Review: check whether another person can rerun your analysis end-to-end.
  • Metric Skepticism: identify misleading metrics and propose better evidence.
  • Bias Reflection: reflect on how data collection/cleaning choices shape results.
Reiterate
  • Rubric-Limited Revision: revise only two rubric criteria; explain why those matter.
  • Peer Review Simulation: respond to reviewer objections with evidence-based edits.
  • Audience Stress Test: re-express for a new audience (and verify meaning holds).
  • Counterargument Re-test: strengthen warrants against strongest objections.
  • Oral Revision Loop: re-record → transcribe → compare transcripts → tighten reasoning.
  • A/B Revision Choice: choose between two revision paths using rubric-based reasons.
  • Change Log: what changed, why, and which objective/rubric row it addresses.
  • Refactor + Tests: improve clarity and add tests; justify tradeoffs.
  • Claim Drift Check: identify any new claims introduced during revision and verify/remove.
  • Case Pack Revision: revise cases and answer keys based on peer performance patterns.
  • Website Usability Pass: revise based on user testing and confusion points.
  • Interactive Refinement: revise checkpoints/feedback for frequent misunderstandings.
  • Media Coherence Pass: align script ↔ visuals ↔ narration; remove contradictions.
  • Accessibility Iteration: improve captions/transcripts/structure/alt-text after review.
  • Versioned Prototype: v1→v2 with a brief rationale video showing improvements.
  • Visual Checklist Loop: revise labels, hierarchy, conventions, and recheck.
  • Phone Re-capture: re-photo the setup/work after revisions; compare v1 vs v2 with notes.
  • Misinterpretation Test: revise to reduce ambiguity and misreads.
  • Infographic Tightening: improve claim–evidence fit; add caveats and sources.
  • Mind Map Cleanup: restructure nodes/links and attach evidence notes to key links.
  • Robustness Iteration: add alternative analyses and document effect on conclusions.
  • Method Swap: compare methods; justify the final choice by assumptions fit.
  • Audit Trail: document exclusions, transformations, and parameters with rationale.
  • Synthetic Data Re-test: vary assumptions in synthetic data and check conclusion sensitivity.
  • Reproducibility Fixes: revise notebook/docs so another person can rerun end-to-end.
