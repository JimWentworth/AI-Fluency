Canvas + DesignPLUS Implementation Guide

Conventions for building certificate pages

Goals

  • Pages render reliably in Canvas.
  • Pages are compatible with DesignPLUS (Cidi Labs) conventions.
  • Interactives that require embeds use responsive iframe wrappers.
  • Code remains maintainable (avoid fragile editor behaviors).

DesignPLUS wrapper (dp-wrapper)

Some DesignPLUS elements require a wrapper on the page to display properly. If a page is missing the wrapper, components may break.

Reference:

  • https://support.cidilabs.com/knowledgebase/how-do-i-add-the-design-tools-wrapper-to-my-page

Project convention: all custom HTML pages include a dp-wrapper at the top-level when using DesignPLUS patterns.

Iframe responsiveness (DesignPLUS wrappers)

DesignPLUS uses wrapper classes for responsive iframes. If the wrapper is removed, styles may not function.

Reference:

  • https://support.cidilabs.com/knowledgebase/why-are-my-iframes-not-displaying-properly

Common wrapper classes (per Cidi Labs guidance):

  • kl_iframe_responsive_scale
  • kl_iframe_fill_width

Project convention: all embeds use the wrapper pattern shown in the Example Canvas Page (07_...).

Editing guidance (avoid breaking DesignPLUS styles)

Using Canvas RCE tooling can sometimes break DesignPLUS styling.

Reference:

  • https://support.cidilabs.com/knowledgebase/why-do-some-designplus-features-break-using-the-canvas-toolbar-and-rich-content-editor

Project convention: treat the Example Page as the “known-good” pattern; minimize manual edits that reflow HTML structure.

Page components (recommended baseline)

  • H2 page title
  • short intro paragraph
  • “Why this matters” bullets
  • key concepts
  • knowledge check / activity
  • embed areas (iframes) only where needed

Implementation checklist (for every page)

  • Uses consistent header hierarchy (H2, H3, H4)
  • Uses dp-wrapper when DesignPLUS components are present
  • Uses iframe wrappers for embeds; includes title attribute
  • Includes alt text / labels as needed
  • Avoids over-nesting and unnecessary inline styles
