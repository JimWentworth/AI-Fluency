AI and the Student Learning Experience (UDL Emphasis)

Learning Cycle: Discover → Understand → Practice → Create → Reflect → Reiterate

This matrix is designed to support Universal Design for Learning (UDL) by expanding multiple means of expression. Each cell lists 4–6 creative, discipline-flexible ways students can use AI to demonstrate learning through varied artifacts (e.g., dialogue, writing/coding, websites, visuals, data stories). Entries are framed as student actions and artifacts rather than tool features. Voice is included when it strengthens learning (oral rehearsal, transcript-based revision, interviews, simulations).

Discover
  • Question Trail: produce a set of curiosity questions to guide reading/viewing.
  • Translation Ladder: rewrite a concept in plain language and back into disciplinary language.
  • Role-Preview: simulate a stakeholder perspective to surface what “matters” in the topic.
  • Micro-Glossary: create a glossary with “confusable pairs” you must distinguish.
  • Voice Warmup: record a 60-second preview, auto-transcribe, annotate unknown terms.
  • Advance Organizer: outline the unit into headings with “what I should be able to do.”
  • Question Funnel: generate broad→narrow research questions with scope constraints.
  • Comparison Scaffold: build a blank comparison table to fill while learning.
  • Starter Code Sketch: draft pseudocode that mirrors a process or system behavior.
  • Note Template: generate a note-taking structure that prompts for evidence and examples.
  • Concept Postcard: one-slide preview (headline, 3 claims, 1 diagram, 1 question).
  • Audio Preview: create a short narrated overview with “things to verify later.”
  • Learning Hub Page: build a simple web page to collect sources, questions, and links.
  • Mini Timeline: create a timeline slide or webpage for key developments.
  • Two-Format Check: compare what changes across text vs audio vs slides.
  • Figure Walk: list observations from a figure before interpretation.
  • Alt-Text Draft: write accessible descriptions that match what is shown.
  • Concept Sketch: draw an initial mental model to refine later.
  • Infographic Plan: draft a layout for a one-claim infographic you’ll populate later.
  • Mind Map Seed: generate nodes for a mind map and label relationship types.
  • Dataset Recon: identify variables, units, missingness, and limits.
  • Question Sorting: generate valid vs invalid questions (with reasons).
  • Data Dictionary: draft variable definitions and assumptions.
  • Bias Radar: list likely biases and how they might affect conclusions.
  • Plan Stub: outline an analysis plan before doing computations.
Understand
  • Socratic Challenge: provide an explanation; answer follow-up “why/how” questions.
  • Example/Non-example Set: create examples and near-misses and justify classifications.
  • Audience Shift: translate the same concept for novice vs expert audiences.
  • Oral Exam Practice: simulate an oral exam; transcribe and annotate reasoning gaps.
  • Misconception Hunt: identify and refute the most tempting wrong interpretation.
  • Three Explanations: novice/technical/metaphor explanations, then critique accuracy and limits.
  • Concept-to-Code: write pseudocode or a short annotated script representing the mechanism.
  • Boundary Tests: generate edge cases and predict outcomes.
  • Explain-Then-Compress: compress an explanation while preserving correctness.
  • Question Bank: create misconception-focused questions with rationales.
  • Mini-Lesson: narrated slides explaining a process (include transcript).
  • Interactive Explainer: build a simple webpage with steps, definitions, and checks.
  • Teach-Back Audio: record → transcribe → revise for clarity and correctness.
  • Storyboard: plan a 60-second explainer video (scenes → claims → evidence).
  • Dual Coding: pair key terms with diagram elements and explain mappings.
  • Observation vs Inference: annotate a figure with observation/inference tags.
  • Diagram Debug: fix missing labels, ambiguous arrows, or incorrect relationships.
  • Two-Model Compare: compare visuals and justify which misleads less.
  • Infographic Critique: evaluate claim–evidence fit and implied meaning.
  • Mind Map Refinement: reorganize nodes into hierarchy and link types.
  • Interpretation Packet: interpret a plot/table + write “what it does NOT show.”
  • Methods Justification: justify why an analysis fits the question (assumptions included).
  • Competing Explanations: generate alternatives and specify what data would decide.
  • Qual Coding Start: draft a coding scheme and justify categories.
  • Unit Check: list what you can explain with evidence and what remains uncertain.
Practice
  • Hint Budget: request one hint at a time after a first attempt; log changes.
  • Step Justification: explain each step; answer probing questions about choices.
  • Role-Play Clinic: simulate a professional conversation and practice response moves.
  • Near-Transfer Set: practice on similar scenarios with new surface features.
  • Error Labeling: classify mistakes and create a targeted repair plan.
  • Practice Variants: generate aligned items with solutions and “common wrong answer” rationales.
  • Fading Scaffolds: worked-example skeletons with increasing blanks.
  • Micro-Code Drills: write small functions/scripts practicing the same logic.
  • Unit Tests: generate tests to validate code or reasoning outputs.
  • Rubric Self-Check: score your attempt against a checklist.
  • Annotated Walkthrough Video: record a solution demo with “why” callouts.
  • Interactive Tutorial: web steps + checkpoints + feedback for common errors.
  • Branching Scenario: decisions with justification prompts and consequences.
  • Demo Site: a small webpage demonstrating a process (toggles encouraged).
  • Oral Rehearsal Loop: speak → transcribe → revise → re-record.
  • Case Image Template: interpret images/figures with a consistent template.
  • Correct vs Incorrect: diagnose which visual feature causes misunderstanding.
  • Checklist Debug: labels, units, hierarchy, conventions, missing components.
  • Mind Map Practice: redraw from memory, compare to reference, repair gaps.
  • One-Claim Infographic: claim + evidence + limitation + source note.
  • Cleaning Log: audit trail + validation checks.
  • Viz Practice: create 3 charts for the same data; justify the best choice.
  • Assumption Checks: list assumptions and run a simple sensitivity test.
  • Notebook Practice: implement analysis steps with narrative explanation.
  • Qual Reliability: code short texts and compare consistency with a peer/rubric.
Create
  • Stakeholder Simulation: test a proposal with a simulated panel (questions + objections).
  • Debate Script: claims/warrants/rebuttals refined for evidence fit.
  • Interview Protocol: draft questions and justify what each probes.
  • Translation Deliverable: produce two audience-targeted versions of the same content.
  • Dialogue-Based Exhibit: Q&A artifact where questions drive the explanation.
  • Paper or Memo: draft and revise using rubric targets.
  • Computational Artifact: prototype/script/notebook demonstrating the concept.
  • Docs Package: README + examples + limitations section.
  • Testing Suite: tests for logic and edge cases.
  • Reusable Workflow: prompt/workflow others can use with constraints explained.
  • Website Deliverable: interactive explainer site or digital exhibit with citations.
  • Micro-Course: short module with media + checks + feedback.
  • Tutorial Series: multi-part walkthrough with checkpoints and error guidance.
  • Digital Story: narrative with embedded audio/visual evidence callouts.
  • Prototype Demo: interactive prototype plus short design-rationale video.
  • Infographic: one finding with evidence, caveats, and sources.
  • Concept Diagram Set: diagram + legend + misread-risk notes.
  • Model Visualization: visual model with assumptions labeled.
  • Synthetic Illustration: labeled synthetic visuals for abstract concepts (transparent).
  • Mind Map Exhibit: publish a mind map with evidence notes for key links.
  • Data Report: question → method → results → limitations → next steps.
  • Reproducible Notebook: analysis steps + outputs + narrative.
  • Dashboard: summary visuals supporting a claim with provenance notes.
  • Robustness Checks: alternate analyses and explanation of conclusion stability.
  • Qual Analysis Memo: themes + excerpts + limitations and bias reflection.
Reflect
  • AI Interview Reflection: an interview-style reflection anchored to specific artifacts.
  • Transcript Annotation: annotate an oral transcript for weak claims/missing evidence.
  • Misconception Autopsy: describe a wrong idea, why it was tempting, and what corrected it.
  • Teaching Script: script how you’d teach the concept and note evidence points.
  • Next-Step Plan: measurable goals + the next artifact you’ll produce.
  • Synthesis Memo: connect concepts and state limits/assumptions explicitly.
  • Notes-to-Reference: transform notes into a compact reference with examples/pitfalls.
  • Revision Rationale: explain which rubric criteria improved and how.
  • Code Reflection: explain each part of your script/notebook and why it’s structured that way.
  • Question Audit: list what you can answer confidently and what remains uncertain.
  • Portfolio Website: curate artifacts with short reflections (“what changed and why”).
  • Narrated Walkthrough: explain decisions/tradeoffs with evidence callouts.
  • Micro-Lesson Revision: revise your interactive tutorial based on user misunderstandings.
  • Accessibility Review: add captions/transcripts/alt-text and reflect on impact.
  • Media Compare: reflect on what you express best in audio vs text vs interactive web.
  • Before/After Visual Audit: identify patterns of improvement across visuals.
  • Misread Analysis: predict viewer misreads and revise to reduce ambiguity.
  • Standards Checklist: personal checklist for future visuals (conventions + clarity).
  • Annotation Reflection: reflect on how labels/legends changed meaning.
  • Transparency Note: document where synthetic visuals were used and why.
  • Error Pattern Report: analyze mistakes and propose targeted interventions.
  • Evidence-of-Change: compare results before/after a strategy with confounds noted.
  • Metric Skepticism: identify misleading metrics and propose better evidence.
  • Reproducibility Review: check if another person can rerun your analysis.
  • Qual Reflexivity Memo: reflect on how bias/position shapes coding and claims.
Reiterate
  • Rubric-Bound Critique: revise two rubric criteria only; document decisions.
  • Peer-Review Simulation: respond to reviewer objections with evidence-based revisions.
  • Audience Stress Test: rewrite for a new audience and verify meaning holds.
  • Counterargument Re-test: strengthen warrants against the strongest objection.
  • Oral Revision Loop: re-record → transcribe → compare transcripts → tighten reasoning.
  • A/B Revision Choice: generate two revision paths; choose with rubric reasons.
  • Change Log: what changed, why, and which objective/rubric row it addresses.
  • Code Refactor: add comments/tests; explain tradeoffs and improvements.
  • Claim Audit: verify/remove any new claims introduced during revision.
  • Conciseness Pass: compress while preserving correctness and evidence linkage.
  • Website Usability Pass: revise based on user testing and confusion points.
  • Media Coherence Pass: align script ↔ visuals ↔ narration; remove contradictions.
  • Interactive Refinement: revise checkpoints/feedback for common misunderstandings.
  • Accessibility Iteration: improve captions/transcripts/structure/alt-text after review.
  • Versioned Prototype: v1→v2 with a brief rationale video.
  • Visual Checklist Loop: revise labels, hierarchy, conventions, and recheck.
  • Misinterpretation Test: revise to reduce ambiguity and misreads.
  • Meaning Constraints: improve design without changing core meaning.
  • Infographic Revision: tighten claim–evidence fit; add caveats and sources.
  • Mind Map Cleanup: restructure nodes/links and add evidence notes for key links.
  • Robustness Iteration: add alternative analyses; document effect on conclusions.
  • Method Swap: compare methods and justify final choice by assumptions fit.
  • Audit Trail: document exclusions, transformations, parameters, and why.
  • Conclusion Stability: state whether conclusions changed and justify.
  • Reproducibility Fixes: revise notebook/docs so someone else can rerun end-to-end.
