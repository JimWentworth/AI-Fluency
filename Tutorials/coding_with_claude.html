Coding with Claude

Use Claude as a partner in reading, writing, and reasoning about code—without outsourcing your understanding or judgment.

Domain · Code

What Does It Mean to “Code with AI”?

Coding with AI is less about having a machine “write programs for you” and more about using a language model as a collaborator. You can ask it to suggest code, explain unfamiliar patterns, refactor for clarity, or generate tests—but you still decide what to run, what to trust, and what to ship.

In learning environments, this makes it possible to get instant feedback and alternative explanations, but it also introduces a risk of “autocomplete learning,” where the model does so much that your own understanding lags behind. The goal is to use AI to extend your reach, not to skip the learning.

Why Use Claude for Code?

Claude is well-suited for reading and reasoning about longer or more complex code. It can:

  • Explain what a piece of code does in plain language.
  • Suggest improvements for clarity, style, or structure.
  • Help generate tests or small example programs.
  • Compare two approaches and discuss tradeoffs.
Good use: paste in a function you don’t fully understand and ask Claude to walk through it line by line, then restate that explanation in your own words.

Quick Start Workflow

When working with Claude on code, try this pattern:

  • Start with a small, clearly scoped task or snippet.
  • Provide context: language, framework, and what you’re trying to do.
  • Ask for reasoning: “Explain your approach step-by-step.”
  • Run and test the code yourself; don’t assume it’s correct.
  • Reflect: what did you learn, and what will you do differently next time?

Example Prompt

I'm learning Python and working with lists.
Here is a function I wrote:

def filter_positive(nums):
    result = []
    for x in nums:
        if x > 0:
            result.append(x)
    return result

1. Explain what this function does, line by line.
2. Suggest a more concise version using list comprehensions.
3. Provide 2-3 test cases I can run to check it.

When to Be Cautious

  • Security: don’t paste proprietary or sensitive code into tools not approved for that purpose.
  • Academic integrity: follow your course’s guidelines on AI assistance.
  • Blind trust: AI-generated code can compile and still be wrong or brittle.
  • Over-reliance: if you can’t explain the code, you don’t really “have” it yet.

Practice Activity

Choose a short function or method (10–30 lines) from your own work or a course example. Then:

  • Ask Claude to explain it in plain language.
  • Ask for one refactored version that improves readability.
  • Ask for a small set of tests or edge cases.
  • Decide which version you prefer and why.
Integrity reminder: if AI significantly shaped your code, document that assistance and be clear about what you wrote versus what was suggested.
